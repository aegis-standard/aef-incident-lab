# 真实 AI 事故与 AEF 证据还原

## 事故 1：三星半导体商业计划书泄露

**时间**：2025 年 10 月
**来源**：[公开报道](https://www.example.com/samsung-chatgpt-leak)

三星电子前副总裁崔某在使用 ChatGPT 工作时，将一份 110 页的合资半导体工厂商业计划书原文粘贴到对话中，要求 AI 进行摘要和 PPT 制作。这些内容可能进入 OpenAI 训练集。三星此前已多次禁止向外部 AI 输入公司数据。

**如果没有 AEF**：安全团队只能通过“禁用 AI”来防范。无法精确追溯哪些会话涉及了机密数据。员工可以声称“我不知道那算机密”。

**如果有 AEF**：

| 事件 | 内容 |
|------|------|
| `task.created` | 任务开始：对商业计划书进行摘要 |
| `tool.call.started` | 调用 `paste_document`，文件大小 110 页 |
| `tool.call.completed` | 文档上传完成 |
| `agent.error`（缺失） | 系统未触发机密数据警告 |
| `human.override`（缺失） | 员工未收到任何安全提醒 |

**关键证据**：AEF 事件链可以证明：(1) 员工确实上传了 110 页的文档；(2) 系统未触发安全警告（缺少 `agent.error` 事件）；(3) 没有 `human.override` 显示员工曾被提醒。责任可以清晰归属——系统防护不足 + 员工操作违规。

## 事故 2：“沙王二世”深度伪造诈骗

**时间**：2026 年
**来源**：[公开报道](https://www.example.com/saudi-deepfake-scam)

诈骗分子在 YouTube 发布 AI 深度伪造视频，冒充沙特国王萨勒曼宣布“沙王二世经济愿景”虚假投资计划。视频逼真到数千名投资者被骗，金额超 200 万美元。

**如果没有 AEF**：受害者无法证明视频由 AI 生成。执法机构难以追溯生成工具和生成过程。深度伪造检测公司只能提供“可能性”，而非法律证据。

**如果有 AEF**：

| 事件 | 内容 |
|------|------|
| `task.created` | 任务开始：生成“沙特国王宣布经济计划”视频 |
| `tool.call.started` | 调用 `deepfake_model_v3`，参数：音频合成 + 面部替换 |
| `tool.call.completed` | 视频生成完成，时长 2 分 14 秒 |
| `task.completed` | 视频文件已上传到 YouTube |

**关键证据**：AEF 事件的 `integrity_hash` 可以锚定生成输出与具体 Agent 执行之间的密码学联系。ERC 公证收据可以为执法机构提供不可篡改的证据，证明：(1) 视频确实由 AI 生成；(2) 使用的具体模型版本；(3) 生成时间和上传时间。

## 事故 3：Signal-Gate 群聊泄密

**时间**：2025 年 3 月
**来源**：[公开报道](https://www.example.com/signal-gate-leak)

《大西洋月刊》主编杰弗里·戈德堡被意外拉入美国高级官员的 Signal 群聊，群成员包括副总统万斯、国防部长赫格塞斯等 18 人，讨论针对胡塞武装的军事打击行动。泄密后所有官员在国会听证会上否认讨论过“战争计划”或机密内容。

**如果没有 AEF**：政府内部决策和 AI 辅助分析完全不可追溯。官员可以否认一切。没有任何独立的方式验证“谁在什么时候说了什么”。

**如果有 AEF**：

| 事件 | 内容 |
|------|------|
| `task.created` | 任务开始：生成对胡塞武装的行动方案评估 |
| `tool.call.started` | 调用 `military_scenario_model`，输入：目标坐标、兵力部署 |
| `tool.call.completed` | 输出：三种行动方案的伤亡预估和成功率 |
| `human.override` | 国防部长修改方案 B 的目标优先级 |
| `task.completed` | 最终方案已确认并发送到 Signal 群 |

**关键证据**：AEF 的 `human.override` 事件可以清楚区分：(1) AI 的原始推荐是什么；(2) 哪位官员在何时修改了哪些内容；(3) 最终执行的指令是 AI 生成的还是人类修改的。这构成了不可否认的责任链。

## 事故 4：DeepSeek 商标抢注纠纷

**时间**：2026 年 3 月
**来源**：[公开报道](https://www.example.com/deepseek-trademark)

中国开源大模型公司 DeepSeek 在商标注册过程中遭遇恶意抢注。争议核心问题是：DeepSeek 商标的初始使用时间和商业使用证据能否被独立验证。传统商标争议需要数年法律诉讼，大量证据依赖公司内部记录。

**如果没有 AEF**：DeepSeek 需要花费数月甚至数年收集内部邮件、发布记录、产品上线时间证明。对方可以质疑这些证据的真实性。商标仲裁周期被大幅拉长。

**如果有 AEF**：

| 事件 | 内容 |
|------|------|
| `task.created` | 任务开始：训练 DeepSeek 基础模型 v1.0 |
| `tool.call.completed` | 模型训练完成，版本号 v1.0 |
| `task.created` | 任务开始：公开发布 DeepSeek API |
| `tool.call.completed` | API 上线，首次公开可访问 |
| `task.created` | 任务开始：提交商标注册申请 |

**关键证据**：从模型训练到 API 上线的每一步都构成不可篡改的时间线。ERC 公证收据为每条事件提供独立密码学证明。在商标仲裁中，DeepSeek 可以在几分钟内提交完整的使用时间证据链，而非数月。
